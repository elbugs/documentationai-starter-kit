Documentación técnica de la API REST de Unified PMS. Todos los endpoints usan el prefijo `/api` y requieren autenticación Bearer Token salvo que se indique lo contrario.

## Autenticación

La API utiliza autenticación mediante **Bearer Token** (JWT).

<Steps>
  <Step title="Obtener token" icon="key">
    Realiza un POST al endpoint de login:

```bash
curl -X POST https://your-app.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "tu-usuario", "password": "tu-contraseña"}'
```
  </Step>
  <Step title="Usar el token" icon="lock">
    Incluye el token en el header `Authorization` de todas las peticiones:

```bash
Authorization: Bearer <token>
```
  </Step>
</Steps>

<Callout kind="info">
  La mayoría de endpoints requieren el parámetro `:pcode` (property code) en la URL para identificar la propiedad.
</Callout>

## Notas generales

- **Property code (`:pcode`):** Identifica la propiedad. Requerido en la mayoría de endpoints.
- **Multi-property (`:pcodes`):** Algunos endpoints aceptan múltiples pcodes separados por coma.
- **Paginación:** Los endpoints de lista aceptan `limit` y `offset` como query params.
- **Filtros de fecha:** Usa `dateFrom` y `dateTo` en formato ISO 8601 (`2026-01-01`).
- **Versionado:** Algunos endpoints tienen versiones alternativas (`/api/v2/...`, `/api/v4/...`).
