## Authentication

The API uses **Bearer Token** authentication. To obtain a token:

<Steps>
  <Step title="Obtener token" icon="key">
    Realiza una petición POST al endpoint de login:

```bash
curl -X POST https://your-app.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "tu@email.com", "password": "tu-contraseña"}'
```
  </Step>
  <Step title="Usar el token" icon="lock">
    Incluye el token en el header `Authorization` de todas las peticiones:

```bash
Authorization: Bearer <token>
```
  </Step>
</Steps>

<Callout kind="info">
  La mayoría de endpoints requieren el header `x-organization-id` para identificar la organización activa.
</Callout>

**Respuesta de login:**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": { "id": "uuid", "email": "tu@email.com", "fullName": "Tu Nombre" }
}
```

## Notas generales

- **Paginación:** La mayoría de endpoints de lista aceptan `limit` y `offset` como query params.
- **Filtros de fecha:** Usa `dateFrom` y `dateTo` en formato ISO 8601 (`2026-01-01`).
- **Organización:** Incluir `x-organization-id` en el header para endpoints que lo requieran.
- **Streaming SSE:** Los endpoints de IA que terminan en `-stream` devuelven Server-Sent Events.
- **Uploads:** Usa `/api/uploads/request-url` para obtener una URL firmada y subir archivos.
