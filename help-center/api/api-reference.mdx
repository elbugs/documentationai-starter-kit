## Autenticación

La API utiliza autenticación mediante **Bearer Token**. Para obtener un token:

<Steps>
  <Step title="Obtener token" icon="key">
    Realiza una petición POST al endpoint de login con tus credenciales:

```bash
curl -X POST https://your-app.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "tu@email.com", "password": "tu-contraseña"}'
```
  </Step>
  <Step title="Usar el token" icon="lock">
    Incluye el token en el header `Authorization` de todas las peticiones:

```bash
Authorization: Bearer <token>
```
  </Step>
</Steps>

## Endpoints principales

### Reservas

| Método | Endpoint | Descripción |
|--------|----------|-------------|
| GET | `/api/bookings` | Lista de reservas |
| GET | `/api/bookings/:id` | Detalle de reserva |
| POST | `/api/bookings` | Crear reserva |
| PATCH | `/api/bookings/:id` | Actualizar reserva |

## Códigos de respuesta

| Código | Significado |
|--------|-------------|
| 200 | OK |
| 201 | Creado |
| 400 | Petición inválida |
| 401 | No autenticado |
| 403 | No autorizado |
| 404 | Recurso no encontrado |
| 500 | Error interno del servidor |

## Notas

- **Paginación:** Los endpoints de lista aceptan `limit` y `offset` como query params.
- **Filtros de fecha:** Usa `dateFrom` y `dateTo` en formato ISO 8601 (`2026-01-01`).
- **Property code:** Incluir `x-pcode` en el header para identificar la propiedad.
